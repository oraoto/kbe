Labels

Kubernetes labels by example

Label（标签）是管理Kubernetes对象的机制。一个Label就是一个键值对，其中会有一些长度和值的限制, 但是它们没有预定义的含义。所以你可以自由选择合适的Label表达你需要的意思，例如可以用Label表达”这个Pod跑在生成环境”或者“这个Pod属于部门X”。

先创建一个最初只有一个Label（env=development）的Pod：

$ kubectl create -f https://raw.githubusercontent.com/mhausenblas/kbe/master/specs/labels/pod.yaml

$ kubectl get pods --show-labels
NAME       READY     STATUS    RESTARTS   AGE    LABELS
labelex    1/1       Running   0          10m    env=development

上面的get pods命令加了--show-labels选项，输出结果会加多一列来显示对象的Label。

你可以给一个Pod加上新的Label：

$ kubectl label pods labelex owner=michael

$ kubectl get pods --show-labels
NAME        READY     STATUS    RESTARTS   AGE    LABELS
labelex     1/1       Running   0          16m    env=development,owner=michael

Label可以用来过滤，例如只列出owner等于micheal的Pod，可以用--selector选项：

$ kubectl get pods --selector owner=michael
NAME      READY     STATUS    RESTARTS   AGE
labelex   1/1       Running   0          27m

--selector选项可以简写成-l，选出env=development的Pod：

$ kubectl get pods -l env=development
NAME      READY     STATUS    RESTARTS   AGE
labelex   1/1       Running   0          27m

通常Kubernetes对象还支持集合选择器。启动另外一个Pod，它有两个Label （env=production 和 owner=michael）：

$ kubectl create -f https://raw.githubusercontent.com/mhausenblas/kbe/master/specs/labels/anotherpod.yaml

列出Label为env=development或env=production的Pod：

$ kubectl get pods -l 'env in (production, development)'
NAME           READY     STATUS    RESTARTS   AGE
labelex        1/1       Running   0          43m
labelexother   1/1       Running   0          3m

Label不仅能用在Pod，实际上你可以把Label应用在各种各样的对象上，例如Node（节点）和Service（服务）。
